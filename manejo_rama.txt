# CASO 1: Isa quiere subir cambios en su rama
git switch rama-isa
git add .
git commit -m "Cambios de Isa"
git push -u origin rama-isa

# CASO 1: Abru quiere subir cambios en su rama
git switch rama-abru
git add .
git commit -m "Cambios de Abru"
git push -u origin rama-abru

/////////////

# CASO 2: Isa quiere ver los cambios de Abru
git fetch
git switch rama-abru
    si no deja: git switch -c rama-abru origin/rama-abru

# CASO 2: Abru quiere ver los cambios de Isa
git fetch
git switch rama-isa
    si no deja: git switch -c rama-isa origin/rama-isa

/////////////

# CASO 3: Isa quiere trabajar en su parte
git switch rama-isa
git pull

# CASO 3: Abru quiere trabajar en su parte
git switch rama-abru
git pull

/////////////

# CASO 4: juntar todo en GitHub
# Hacen dos Pull Requests:

rama-isa → main
rama-abru → main





Caso 1: ella editó y quiere subirlo
En la compu de ella:

git add .
git commit -m "Cambios de amiga"
git push -u origin rama-amiga

Ahora esos cambios están en GitHub.

Caso 2: vos querés ver los cambios de ella
En tu compu:

git fetch
git switch rama-amiga

Si no te deja:
git switch -c rama-amiga origin/rama-amiga

Caso 3: vos querés trabajar en tu parte
En tu compu:

git switch main
git pull
git switch -c mi-rama

Trabajás, y después:

git add .
git commit -m "Mis cambios"
git push -u origin mi-rama


Caso 4: quieren juntar todo
Van a GitHub y hacen Pull Request:
rama-amiga → main
mi-rama → main
