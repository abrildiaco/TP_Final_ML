# 5. modelos

1. Regresión Lineal
  Se utiliza como modelo base o baseline. Permite tener una primera referencia del error predictivo y analizar si existe una relación aproximadamente lineal entre las variables del vehículo y el precio.


2. Ridge Regression
  Se incluye como una extensión de la regresión lineal con regularización L2. Es útil porque, luego del one-hot encoding de variables categóricas como marca, modelo y color, el dataset puede tener muchas columnas y posibles correlaciones entre variables.
  La regularización ayuda a controlar coeficientes muy grandes y reduce el riesgo de overfitting.

3. LASSO Regression
  Se utiliza porque incorpora regularización L1, que puede llevar algunos coeficientes exactamente a cero. Esto permite hacer una especie de selección automática de variables.
  Es útil para analizar qué variables parecen aportar más información al precio y para obtener un modelo lineal más simple.

4. Random Forest Regressor
  Se utiliza porque combina muchos árboles de decisión mediante bagging. Es un modelo capaz de capturar relaciones no lineales e interacciones entre variables sin tener que especificarlas manualmente.
  Además, es robusto frente a outliers y permite analizar la importancia de las variables.

5. Gradient Boosting Regressor
  Se incluye porque los modelos de boosting construyen árboles de manera secuencial, donde cada nuevo árbol intenta corregir los errores del modelo anterior.
  Es especialmente adecuado para datos tabulares y suele lograr muy buen rendimiento predictivo en problemas de regresión. Por eso es uno de los principales candidatos a obtener el menor RMSE.

6. Red Neuronal / MLP Regressor
  Se utiliza como modelo no lineal más flexible. Una red neuronal puede aprender relaciones complejas entre las variables de entrada y el precio.
  Sin embargo, requiere normalización de variables, ajuste cuidadoso de hiperparámetros y control de overfitting mediante regularización o early stopping. Se incluye principalmente para comparar su desempeño frente a modelos clásicos de machine learning.

## Criterio de comparación
  Todos los modelos se evaluarán con métricas de regresión, principalmente RMSE, ya que es la métrica indicada en la consigna para la evaluación final. También se pueden reportar MAE y R² para complementar el análisis.
  La comparación entre modelos permitirá observar si modelos simples sufren underfitting, si modelos más complejos logran capturar mejor la relación entre las variables y el precio, y si alguno presenta señales de overfitting.
